# 実験レポート自動化ツール プロジェクト計画書

## 1. はじめに

本ドキュメントは、理系大学生の実験レポート作成を効率化・自動化するWebアプリケーション(以下、「本ツール」)の開発プロジェクトに関する計画を定めるものです。プロジェクトの目的、範囲、システム構成、開発計画、リスク管理などを定義し、関係者間の共通理解を形成することを目的とします。

## 2. プロジェクトの目的と目標

### 2.1. 目的

本プロジェクトの主な目的は、AI技術を活用して、理系大学生が実験レポート作成に費やす時間を大幅に削減し、より本質的な学習活動である「考察」や「分析」に集中できる環境を提供することです。

### 2.2. 目標

- **時間短縮**: レポート作成にかかる平均時間を、現状の10〜12時間から2〜3時間へと80%削減する。
- **品質向上**: AIによるデータ分析(統計処理、グラフ生成、傾向分析)を通じて、手作業によるミスを減らし、レポートの客観性と正確性を向上させる。
- **学習支援**: 考察部分の自動生成は行わず、学生自身が思考を深めるための「基盤」を提供することで、倫理観を尊重しつつ学習効果を最大化する。
- **利便性**: 過去のレポート資産を再利用できる機能を提供し、継続的なレポート作成の効率化を支援する。

## 3. プロジェクトの範囲

### 3.1. 主な機能(スコープ内)

| 機能カテゴリ | 主要機能 |
| :--- | :--- |
| **ユーザー管理** | ユーザー登録、ログイン、パスワード管理 |
| **データ入力** | 実験データ(Excel, 写真, コード)のアップロード機能 |
| | 過去レポート(Word)をアップロードし、基盤を抽出する機能 |
| **AI分析** | Dify APIと連携した統計処理、グラフ生成、傾向分析、理論値比較 |
| **レポート生成** | AI分析結果をJinja2テンプレートに適用し、Word(.docx)形式でレポートを自動生成 |
| | 考察部分は学生が記入するためのテンプレート(基盤)のみ生成 |
| **参考文献** | 関連する参考文献の自動検索およびリスト生成機能 |
| **履歴管理** | 生成したレポートのデータベース保存、閲覧、ダウンロード機能 |

### 3.2. スコープ外

- リアルタイム共同編集機能
- モバイル専用アプリケーションの開発(初期バージョンではレスポンシブWebデザインで対応)
- Dify API自体の開発(既存のAPIを活用)

## 4. システムアーキテクチャ

本ツールは、モダンなWebアプリケーションの標準構成を採用します。

- **フロントエンド**: ユーザーが直接操作するインターフェース。React.jsやVue.jsなどのJavaScriptフレームワークを使用して構築します。
- **バックエンド**: ビジネスロジックを処理するサーバーサイドアプリケーション。PythonのFastAPIまたはFlaskフレームワークを使用し、Dify APIとの連携、Word文書生成、データベース操作などを担当します。
- **データベース**: ユーザー情報、レポート履歴などを永続的に保存します。PostgreSQLやMySQLなどのリレーショナルデータベースを採用します。
- **外部API**: Dify API(AI分析)、参考文献検索APIを活用します。

## 5. 開発計画

本プロジェクトは、アジャイル開発のアプローチを取り入れ、以下の5つのフェーズで進行します。

| フェーズ | 主なタスク | 期間(目安) |
| :--- | :--- | :--- |
| **フェーズ1: 基盤構築** | 開発環境設定、データベース設計、ユーザー認証機能の実装、基本的なUI/UXの構築 | 2週間 |
| **フェーズ2: コア機能実装** | データアップロード、Dify API連携、Word文書生成機能の実装 | 3週間 |
| **フェーズ3: 高度な機能実装** | 過去レポート活用機能、参考文献自動検索、レポート履歴管理機能の実装 | 2週間 |
| **フェーズ4: テストと改善** | 単体テスト、統合テスト、ユーザーテスト(UAT)の実施、バグ修正 | 1週間 |
| **フェーズ5: デプロイと運用** | 本番環境へのデプロイ、ドキュメント整備、運用監視体制の構築 | 1週間 |

**合計期間(目安): 9週間**

## 6. 技術スタック(推奨)

| 領域 | 技術 | 理由 |
| :--- | :--- | :--- |
| **フロントエンド** | React.js / Vue.js | 豊富なライブラリとコミュニティがあり、効率的なUI開発が可能。 |
| **バックエンド** | FastAPI (Python) | 高速なパフォーマンスとPythonの豊富な科学技術計算ライブラリとの親和性。 |
| **データベース** | PostgreSQL | 高機能かつ信頼性の高いオープンソースRDBMS。 |
| **認証** | JWT (JSON Web Token) | ステートレスな認証方式で、スケーラビリティに優れる。 |
| **Word生成** | `python-docx` | PythonでWord文書を柔軟に操作できる標準的なライブラリ。 |
| **インフラ** | Docker, AWS/GCP | コンテナ化による環境差異の吸収と、クラウドによる柔軟なスケールが可能。 |

## 7. リスク管理

| リスク | 発生可能性 | 影響度 | 対策 |
| :--- | :--- | :--- | :--- |
| **Dify APIの性能問題** | 中 | 大 | 非同期処理を導入し、バックグラウンドでジョブを実行。ユーザーには進捗を通知する。 |
| **Word生成の再現性** | 中 | 中 | `python-docx`で対応できない複雑な書式は、テンプレート側を簡略化する方向で調整。 |
| **データセキュリティ** | 低 | 大 | 全ての通信をHTTPS化し、データベースを暗号化。アクセス制御を徹底する。 |
| **スコープの肥大化** | 高 | 中 | 各フェーズの開始時に要件を再確認し、スコープ外の要求は次期バージョンで検討する。 |

## 8. 成功の評価指標

- **定量的指標**: レポート作成時間の削減率、月間アクティブユーザー数(MAU)、システムの稼働率。
- **定性的指標**: ユーザー満足度アンケートの結果、SNSや学内での口コミ評価。
