# 再デプロイの手順

このプロジェクトはVercelにデプロイされていると想定されます。再デプロイを行うには、以下のいずれかの方法を実行します。

## 1. 変更がある場合 (通常の手順)

コードに変更を加えた後、Gitでコミットしてプッシュすると、Vercelが自動的に検知してデプロイを開始します。

```bash
git add .
git commit -m "変更内容のメッセージ"
git push
```

## 2. 変更がない場合 (手動再デプロイ)

コードに変更はないが、再デプロイを行いたい場合（例：環境変数の変更を反映させたい場合など）は、空のコミットを作成してプッシュします。

```bash
git commit --allow-empty -m "Trigger redeploy"
git push
```

## 3. Vercel CLIを使用する場合 (上級者向け)

Vercel CLIがインストールされている場合は、以下のコマンドで手動デプロイが可能です。

```bash
vercel --prod
```
